↑ KO call and parallel actions fire simultaneously the moment the deal closes

✅ Step 1 — Deal Closes (Salesforce Closed-Won) · T=0
  • Salesforce Closed-Won triggers the onboarding pipeline
  • Two parallel workstreams fire immediately
🔄 Step 2 — Parallel: Instance Creation + Intelligence Scrape · Immediate

These two workstreams run simultaneously — neither waits for the other, and neither waits for the kickoff call.

🖥️ Workstream A
Instance Creation
  • Instance requested immediately on close — same-day SLA
  • Scrape output feeds directly into instance configuration
  • Customer record, contacts, users pre-loaded from SF + Outreach data
  • Integration flags, invoice settings, dispatch model applied where known
  • SA context note created in system with full client brief
  • Client arrives to an instance that already knows who they are
🧠 Workstream B
Sales Call + Salesforce Scrape
  • Outreach transcripts pulled by company name — all sales calls
  • Salesforce Account, Opportunity, Contact records extracted
  • Signals extracted: prior PSA, accounting SW, RMM, contacts, pain points, commitments, migration concerns, integration priorities
  • Client Brief auto-populated → loaded to Notion client page
  • Gaps flagged — follow-up questions identified for KO call
  • SA receives brief before the kickoff call fires
📞 Step 3 — Scaled-Down Kickoff Call · Within 5 Hours of Close

This is no longer a discovery call. The SA already has the Client Brief. The KO call has one job: give the client their deliverables and schedule the Jump Start Sessions.

  • Not about collecting information — that came from the sales cycle scrape. SA confirms and fills any gaps the scrape couldn't answer.
  • Client receives their deliverables — data import CSV templates, pre-kickoff checklist, any pre-work needed before Jump Start 1
  • Data import clock starts now — client begins cleanup in parallel, not mid-engagement
  • Jump Start Sessions scheduled — 2 sessions confirmed for the next 3–4 days
  • Call is short — no re-running the sales discovery process
📤 Client Deliverables Issued at KO
  • Data import CSV templates (customers, contacts, products, ticket history)
  • Pre-work checklist for Jump Start 1 (GL codes, user list with emails, integration credentials)
  • Jump Start Session calendar invites — confirmed dates in next 3–4 days
⚡ Step 4 — Jump Start Session 1 · 3 Hours · Days 1–2

Deep-dive working session. The Client Brief is on screen. SA and client work through it together — confirming what was auto-configured, completing what's still open, and pivoting to platform training as fast as possible.

Hour 1
Client Brief Review + Configuration Completion
Walk through the Notion Client Brief together. Confirm what was pre-configured from the scrape. Complete remaining configuration items — anything the scrape flagged as a gap. Users, roles, integrations, dispatch board, billing setup.
Hour 2
Core Platform Training — Ticketing + Workflows
Begin platform training with configuration mostly complete. Ticket creation, service board workflow, assignment and dispatch. Client hands on the platform in their configured environment — not a generic demo.
Hour 3
Billing + Reporting + Client Portal
Billing workflow, invoice generation, payment setup, customer portal configuration. SA confirms data import status — how far along is client with CSV cleanup?
Session Outputs
  • Configuration items completed or assigned with owner + due date
  • Client comfort level assessed — what do they need more time on in JS2?
  • Data import status confirmed — ETA for clean CSV
⚡ Step 5 — Jump Start Session 2 · 3 Hours · Days 3–4

Complete the configuration and training. Session ends with a formal Operational Readiness assessment using the Onboarding Tracker — SA determines what's needed to get to go-live and plans accordingly.

Hour 1
Complete Open Configuration + Advanced Features
Any items outstanding from JS1. Advanced configuration: SLA policies, automation rules, integrations that needed credentials. SMS setup, email connector if not already live.
Hour 2
Platform Deep-Dive Training — Client-Specific Workflows
Training tailored to their vertical and service model. For MSPs: RMM-to-ticket workflows, recurring managed service billing. For security integrators: install project management, dispatch board, inventory. For mixed shops: both tracks.
Hour 3
Operational Readiness Assessment + Go-Live Plan
SA formally assesses health using the Onboarding Tracker and Checklist. Scores each milestone area: what's complete, what's in-flight, what's blocked. Plans accordingly — determines if client goes live, needs additional sessions, or has open items that can be resolved async.
Readiness Assessment Outputs
  • Onboarding Tracker status: completed milestones vs. open items
  • Go-live date confirmed or revised based on complexity tier
  • Remaining open items assigned: owner, due date, follow-up cadence
  • Data import status — if not received, escalation path triggered
  • Client confidence score — SA judgment on readiness to operate independently
🚀 Step 6 — Post-Jump Start: Go-Live or Structured Follow-Through
  • Simple clients (no data import/migration, no token migration, no integrations): live within 1 week — Jump Starts complete the engagement
  • Medium and complex clients: SA plans remaining sessions or async work based on Readiness Assessment from end of JS2
  • Data import processed by data engineering once client delivers clean CSV — timeline depends on client data delivery
  • If client has not delivered CSV by JS2: structured follow-up cadence begins (Day 0 call → Day 2 email → Day 4 escalate) — not open-ended
  • Status tracked in Onboarding Tracker: On Track / Off Track / On Hold (with hold date) / RTS

🤖 What the Automated Pipeline Does on Closed-Won
⚙️ Instance Auto-Configuration
Engineering handles fallout only
  • Instance created automatically
  • Outreach + SF scrape runs immediately
  • Signals extracted and applied: customer record, contacts, users, integration flags, invoice settings, dispatch config

📧 Client Communication
Automated — no SA involvement
  • Welcome email sent automatically on Closed-Won
  • Includes short video walkthrough of the platform and what to expect

  • Data import CSV templates attached — cleanup starts immediately
  • Pre-work checklist included — what to bring to Jump Start 1
  • Login credentials for their pre-configured instance
🎯 What Changes for the SA in Near-Term

The SA enters the engagement at Jump Start 1 — not at kickoff. They've already reviewed the Notion Client Brief before the session. The instance is pre-configured. The client has been walking around in their environment since they received credentials. Jump Start 1 opens with: "Here's what we built for you. Here's what we still need to complete together. Let's go."

SA time is spent entirely on the work that requires a human — completing complex configuration items, training the client on their specific workflows, and making judgment calls on edge cases. Not re-collecting information. Not building from scratch.

Section 04
⏱️ Go-Live Duration Tiers

Client complexity determines go-live timeline. These targets assume Jump Start Sessions are completed in days 1–4. Data import timelines are the primary variable for Medium and Complex tiers.

🟢
Simple
1 Week
  • No data import or migration required
  • No token migration
  • No third-party integrations
Does not apply if:
  • Any data import from a prior system
  • Token migration required
  • Integration connections needed
🔵
Medium
3 Weeks
  • Data import from 1 system or migration
  • Token migration
  • Calendar integration (Google or Outlook)
  • Teams or Slack integration
Escalates to Complex if:
  • More than 1 source system to import
  • All other integrations
🟡
Complex
6 Weeks
  • Data import from more than 1 system
  • Token migration required
  • Multiple integrations
  • Non-standard or custom integrations
  • Multi-location / parent-child structure
Duration note:
  • Upper bound driven by client data delivery speed
  • SA assesses at end of JS2 and plans remaining timeline
📌 The Data Import Variable

For Medium and Complex tiers, the primary duration driver is how fast the client delivers clean CSV data. In the short-term model, the data import clock starts at the kickoff call (day 0). In the near-term model, it starts when the Closed-Won email fires. Either way, it's weeks earlier than today — where the clock starts mid-engagement when the SA first mentions the format requirement.

Clients who deliver clean data fast will land at the low end of their tier. Clients who drag out cleanup extend their own go-live. The structured Hold/RTS cadence ensures this doesn't quietly compound into a 60-day engagement.
