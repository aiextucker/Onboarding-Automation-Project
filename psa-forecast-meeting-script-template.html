PSA Weekly Forecast Call — SA Script Template

_Source transcript: Fireflies PSA Web Weekly Forecast (01KQTXD30093B662PCRNX0JGPG), 2026-05-12._

What the transcript says the script needs to solve

The call was too anecdotal and inconsistent across SAs. Evan specifically asked for a formalized script so every SA update sounds the same, moves quickly, and gives leadership an objective read on where each project actually stands.

Key requirements from the transcript:

  • Start with movement from last week: graduations, RTS, on-hold movement, active count.
  • Make updates objective: checklist/phases complete, percent complete, what is done, what is next, what is blocked.
  • Avoid rambling client-by-client narratives.
  • Force a specific forecast date, especially for month-end forecasts.
  • If blocked, name the blocker, owner, and strategy to clear it.
  • Make sales/accountability visible, especially for RTS, on-hold, cancellations, and expectation mismatches.
  • Keep client interactions current; stale activity creates false risk signals.
  • Handle RTS/on-hold with discipline: if Sales cannot re-engage quickly, move toward pre-cancel/cancel.

1. Facilitator Opening — 2 minutes

“Today we’re going to run this as an objective forecast review. We’ll start with movement since last week, then each SA will use the same format: forecast date, percent complete, completed phases, next phase, blockers, owner, and confidence. If the update does not change the forecast, say that clearly. If the forecast is at risk, name the decision or escalation needed before we move on.”

Roll-up metrics to open with

  • Active onboarding count: [count]
  • On hold count: [count]
  • RTS count: [count]
  • Graduated since last forecast: [count / clients / MRR]
  • Moved to RTS since last forecast: [count / clients / MRR]
  • Coming out of RTS / re-engaged by Sales: [count / clients]
  • Month forecast target: [target]
  • Currently credible for this month: [count]
  • Forecasts sitting on last day of month: [count]
“The key question today is not ‘what happened?’ It is: what will actually graduate, what is slipping, and what needs help?”

3. Forecast Confidence Language

High confidence

Use when the client is engaged, the next meeting is scheduled, blockers are minor/owned, and graduation criteria are mostly complete.

“High confidence. The forecast date is still good. Remaining work is [specific remaining work], and it is scheduled for [date].”

Medium confidence

Use when the client can still graduate, but there is a known dependency.

“Medium confidence. I’m holding the forecast for now, but it depends on [dependency] being resolved by [date]. If that misses, I recommend moving the forecast to [new date].”

Low confidence

Use when the forecast date is mostly aspirational, the client is not engaged, or the blocker has no firm resolution path.

“Low confidence. I do not think [date] is credible unless [specific thing] happens by [date]. My recommendation is [move to new date / RTS / escalate].”

4. Blocker Categories and Script Language

Product / engineering blocker

Examples from transcript: PO/work-performed token gaps, user group assignment notifications, Quick Copper 403 permissions, workflow/editor issues.

“This is blocked by a product/engineering issue: [issue]. Owner is [person/team]. Client impact is [impact]. Strategy to clear is [ticket/escalation/workaround]. Needed by [date] to protect the forecast.”

Client engagement blocker

Examples: no-shows, low activity, pushing meetings, unresponsive clients.

“This is blocked by client engagement. Last meaningful contact was [date]. We have attempted [attempts]. Sales rep is [rep]. Recommendation is [keep active with next meeting / move to RTS / pre-cancel].”

Sales expectation / contract blocker

Examples: Red Gap contract confusion, trial/non-payment, client thought PSA was included.

“This is a Sales/expectation issue. Client understanding is [what they believe]; contract/payment reality is [actual state]. Sales owner is [rep]. Recommendation is [Sales re-engage / RTS / pre-cancel] by [date].”

Data/import blocker

Examples: TigerPaw API keys, missing product fields, bulk tool gaps.

“This is blocked by data/import readiness. Missing item is [data/API/key/template field]. Owner is [client/data team/SA]. ETA is [date]. Forecast impact is [none / at risk / move date].”

Process/training blocker

Examples: client wants more internal testing before support handoff; final walkthrough/Q&A pending.

“Training/configuration is mostly complete. Remaining item is [walkthrough/testing/Q&A]. Client needs [specific session/action] before graduation. Forecast confidence is [level].”

5. RTS / On-Hold Discipline

Use this when a client is stalled, on hold, or ready-to-schedule but not engaging.

On hold / RTS readout

“[Client] is currently [On Hold / RTS] because [reason]. Sales rep is [rep]. Last client activity was [date]. Re-engagement owner is [owner]. If not cleared by [date], recommendation is [pre-cancel / cancel / keep RTS with reason].”

Suggested operating rule from transcript

  • Sales gets roughly 2–3 weeks to clear RTS/on-hold re-engagement.
  • If not cleared, move to pre-cancel.
  • If still not cleared after the pre-cancel window, proceed to cancellation.

Script

“This has exceeded the re-engagement window. Unless Sales confirms movement by [date], I recommend moving to pre-cancel.”

6. Month-End Forecast Pressure Check

Use this when many clients are forecast on the last day of the month.

“This forecast is currently sitting on [month-end date]. Is that a real graduation date or a placeholder?”

If real:

“What exact work happens between now and then, and on what dates?”

If placeholder:

“Let’s replace it with a real date or mark confidence low.”

Use these to keep the meeting tight.

  • “What percent complete are they, based on milestones?”
  • “What phases are done?”
  • “What is next?”
  • “What is blocking graduation?”

8. Example Update Formats

Clean / on-track client

“Prestige is forecast for May 28. Confidence is high. They are 90% complete: billing, products, and ticketing are done. Final walkthrough and Q&A are scheduled for May 22. No blockers. Recommendation is to keep the date. Halle owns the walkthrough and will confirm graduation readiness after that call.”

At-risk client

“Quick Copper is forecast for May 31. Confidence is medium-low. They are blocked by 403 permission errors when techs create contacts. Mark owns the client follow-up; Ray/engineering owns root-cause tracing. If we do not resolve permissions by May 20, I recommend moving graduation into June.”

RTS candidate

“Atlantic is not forecastable right now. The client keeps pushing meetings because their acquisition dependency is not ready. Sales rep is [rep]. Recommendation is RTS and Sales re-engagement. If no movement in two weeks, move to pre-cancel.”

Product workaround client

“VanRan is blocked by subcontractor access needs. Current workaround is one portal user plus tagging, with no extra billing for those technicians while Product works the correct model. Halle owns client communication; Ray owns the permission/template recommendation. Confidence is medium until the client accepts the workaround.”

9. Closing Script — 3 minutes

“Before we close, let’s confirm: which dates changed, which clients are no longer credible for this month, which blockers need escalation, and which clients need Sales re-engagement.”

Closeout checklist

  • Forecast date changes captured.
  • RTS/on-hold/pre-cancel decisions captured.
  • Blockers have owner + due date.
  • Sales rep visible for RTS/on-hold/cancel-risk clients.
  • Client interactions updated for stale accounts.
  • Product/engineering escalations assigned.
  • Month-end placeholders converted to real dates or low confidence.
“The output of this meeting should be a believable forecast, not a narrated dashboard.”
