Section 03
🔑 Brief Section 2 — The Core 8

The 8 data points every SA needs at kickoff. This is where Salesforce + transcripts together can get to 80–90% coverage.

Core 8 Item SF Field Notes / Fallback Status
1️⃣ Prior PSA / Current System Account.PSA_Platform__c Picklist — well-populated. Fallback: transcript scrape for PSA mentions. Auto
2️⃣ Accounting Software Account.Accounting_Platform__c / Account.Billing_Platform__c Two fields — both should be checked. QBO vs Desktop matters for integration. Auto
3️⃣ RMM Tool Account.RMM_Platform__c + Opp.Are_they_using_RMM__c Picklist exists. Boolean confirms whether in use. Supplement from transcript. Partial
4️⃣ Payment Processor Account.Payment_Processor1__c (picklist) + Account.Payment_Processor__c (text) Two fields — check both. Activity thread is often the richest source (as with NJOS). Enrich
5️⃣ User Count + Roles Opportunity.User_Count__c + Account.NumberOfEmployees Count available. Roles/titles come from Contact records. Job titles should be scraped from transcript. Partial
6️⃣ Dispatch Model + Business Hours — Not captured in SF. Must come from transcript or Sales follow-up questionnaire. Enrich
7️⃣ Invoice Number Sequence / Continuity — Not in SF. Transcript often mentions it. Flag as follow-up if not found. Enrich
8️⃣ SureTax — existing or new? Account.Tax_database__c Picklist exists. Check if populated. Supplement from transcript. Partial

Section 07
🔴 Salesforce Gap Analysis — What Sales Needs to Own

These fields exist in Salesforce but are routinely blank at close. Requiring them before Closed-Won is the single highest-leverage process change Sales can make.

⚠️ Fields to Require at Closed-Won (Salesforce Validation Rule)
  • Opportunity.Requires_Data_Conversion__c — determines Simple vs Medium vs Complex tier. Blank = SA can't plan the engagement.
  • Opportunity.Kickoff_Call_Date__c — should be set at close so the SA calendar hold triggers automatically.
  • Opportunity.Rev_io_Edition__c — which billing package was sold. SA needs this for instance configuration.
  • Opportunity.X1_Concern_with_Rev_io__c — top objection. SA walks in knowing what to address proactively.
  • Opportunity.X1_Impressive_Feature__c — what sold them. SA can lean into this to build momentum in Jump Start 1.
  • Account.RMM_Platform__c — required for integration planning. Often blank even when RMM is confirmed in conversation.
💡 Transcript as the Safety Net

When SF fields are blank, the Outreach scrape is the fallback. The transcript pipeline should flag any brief section where SF returned null and attempt to extract the answer from sales call audio. If transcript extraction also fails, those items become the 5–7 follow-up questions sent to the Sales rep automatically before kickoff.

Section 08
📊 Summary — What Auto-Populates vs. What Doesn't
✅ Auto-populates from Salesforce
  • ✓ Company name, location, phone, website
  • ✓ MRR, close date, one-time fees
  • ✓ Prior PSA (PSA_Platform__c)
  • ✓ Accounting platform / QuickBooks flag
  • ✓ Token migration required (boolean)
  • ✓ International requirements (boolean)
  • ✓ User count (two fields — use both)
  • ✓ Vertical / industry
  • ✓ Company size / employee count
  • ✓ TigerPaw migration flag
  • ✓ Existing billing / TP client flags
  • ✓ Business issues (when filled)
  • ✓ Contacts — name, title, email
  • ✓ QB and ConnectWise integration booleans
⚡ Enrich from transcript
  • → Payment processor (supplement SF)
  • → Dispatch model + business hours
  • → Invoice number / continuity preference
  • → SMS, inventory, projects, quoting signals
  • → Migration concerns / red flags
  • → Commitments made by Sales
  • → User roles and job titles
🔴 Sales must fill at close
  • ! Data conversion required?
  • ! Kickoff call date
  • ! Rev.io Edition / package
  • ! #1 concern with Rev.io
  • ! #1 impressive feature
  • ! RMM platform (if using RMM)
